# Security Policy

## Supported Versions

安全修复以默认分支上的最新代码为准，历史快照和本地导出数据不承诺持续维护。

## What to Report

欢迎报告以下问题：

- 命令执行、路径处理、文件覆盖等可能带来的安全风险
- 下载流程中的不安全默认行为
- 依赖或自动化流程中明显可利用的问题

以下内容通常不属于本仓库的安全问题范围：

- FDA 官网内容本身的问题
- 你本地下载到的数据质量问题
- 因错误分类规则导致的业务归档偏差

## Reporting a Vulnerability

如果 GitHub 已为该仓库启用 private vulnerability reporting，请优先使用该渠道。

如果暂时没有私密报告入口，请不要公开披露可利用细节。可以先提交一个不包含敏感细节的 issue，请求维护者建立私下沟通渠道后再补充信息。

## Response Expectations

维护者会尽力确认问题、评估影响，并在可行时提供修复或缓解建议；但该仓库目前按 best-effort 方式维护，不承诺企业级 SLA。
